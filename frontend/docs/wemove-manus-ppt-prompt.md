# Manus AI 발표 자료 생성 프롬프트

아래 조건으로 한국어 발표용 PPT를 만들어줘.

## 프로젝트 정보

- 서비스명: WeMove
- 서비스 성격: 지역 기반 운동 모임 플랫폼
- 핵심 설명: 사용자가 지역, 종목, 일정 기준으로 운동 모임을 찾고, 만들고, 참여하고, 관리할 수 있는 웹 서비스
- 발표 대상: 교수, 실무 면접관, 포트폴리오 심사자
- 목표: 학생 과제 티보다 실제 서비스 기획/개발 발표처럼 보이게 만들기

## 디자인 방향

- 전체 톤은 세련되고 실무 발표 느낌으로 구성
- AI가 자동 생성한 흔한 템플릿 느낌은 최대한 줄여줘
- 화이트와 연한 블루를 기본으로 하고, 포인트 컬러는 선명한 블루를 사용해줘
- 카드형 UI, 정돈된 여백, 정보 위계가 잘 보이게 구성해줘
- 과한 3D 아이콘, 추상 배경, 불필요한 장식은 빼줘
- 실제 SaaS 서비스 발표 자료처럼 보이게 해줘

## 슬라이드 구성 원칙

- 슬라이드 수는 25~35장 정도
- 각 슬라이드는 텍스트를 너무 많이 넣지 말고 핵심만 정리
- 기능 소개 슬라이드에는 오른쪽 또는 아래에 스크린샷 삽입 공간을 남겨줘
- 스크린샷 자리에는 반드시 다음 문구를 플레이스홀더로 넣어줘:
  - `[시연 이미지 삽입]`
- 표지, 문제 정의, 해결 방식, 주요 기능, 관리자 기능, 신고 기능, 배포, 트러블슈팅, 개선 방향까지 포함해줘

## 주요 기능 설명에 반영할 요소

- 홈에서 인기 모임과 이번 주 일정, 최근 활동을 확인할 수 있음
- 인기 모임은 오늘 / 최근 7일 토글 가능
- 모집 완료 / 취소 모임은 인기 모임에서 제외
- 모임 찾기에서 썸네일, 일정, 위치, 종목, 모집 상태를 빠르게 확인 가능
- 내 활동 페이지에서 참여 예정, 참여 대기, 내가 만든 모임, 참여 완료, 캘린더, 최근 활동을 통합 제공
- 신고 시 사유 선택과 직접 입력 사유를 구분해서 보여줌
- 관리자 페이지에서 회원, 모임, 신고, 종목을 관리함
- 사용자가 정지되면 모임원은 자동 제외, 모임장이면 모임 자동 취소 처리
- 진행 중 모임은 24시간이 지나면 자동 완료 처리
- Vercel + Render 분리 배포 구조를 사용함
- 배포 과정에서 로그인 유지, 쿠키 설정, 이미지 경로 이슈를 해결함

## 기술 스택

- Frontend: React, Vite, React Router, Axios
- Backend: Spring Boot, MyBatis
- Database: MySQL
- Cache: Redis
- Image Storage: Cloudinary
- Deployment: Vercel, Render

## 슬라이드 문체

- 대학 과제 발표처럼 가볍지 않게
- 그렇다고 지나치게 마케팅스럽지도 않게
- 개발자가 직접 만들고 개선한 서비스 발표처럼 자연스럽게
- 문장은 짧고 명확하게
- 불필요한 영어 남발 금지

## 추가 요청

- 각 슬라이드에 제목과 3~5개의 핵심 bullet을 넣어줘
- 스크린샷이 들어가면 좋은 슬라이드는 `[시연 이미지 삽입]` 플레이스홀더를 배치해줘
- 마지막에 질의응답 슬라이드도 1장 추가해줘
