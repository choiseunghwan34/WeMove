# WeMove 발표용 PPT 상세 원고

## 0. 문서 사용법

- 이 문서는 단순 목차가 아니라 발표 슬라이드에 바로 옮길 수 있는 상세 원고입니다.
- 각 슬라이드는 `제목`, `슬라이드 본문`, `발표 포인트`, `시연 이미지 자리`로 구성했습니다.
- 분량은 35~45장 기준으로 잡았고, 필요하면 일부 슬라이드를 합쳐서 25장 안팎으로 줄일 수 있습니다.
- 발표 톤은 학생 과제 느낌보다 실제 서비스 설계 발표 느낌에 가깝게 작성했습니다.

## 1. 표지

### 슬라이드 제목
WeMove

### 슬라이드 본문
- 지역 기반 운동 모임 플랫폼
- 운동을 더 쉽게 시작하고 더 오래 이어갈 수 있도록 설계한 웹 서비스
- 사용자가 모임을 찾고, 만들고, 참여하고, 관리할 수 있는 구조 구현

### 발표 포인트
- 단순 게시판이 아니라 운동 모임의 탐색부터 운영까지 연결하는 서비스라는 점을 먼저 강조합니다.
- 서비스 이름과 함께 브랜드 이미지가 전달되도록 메인 홈 화면을 배경으로 두는 것이 좋습니다.

## 2. 프로젝트 개요

### 슬라이드 제목
프로젝트 개요

### 슬라이드 본문
- 프로젝트명: WeMove
- 프로젝트 유형: 커뮤니티형 운동 모임 서비스
- 주요 사용자: 일반 사용자, 모임장, 관리자
- 목표: 지역 기반 운동 모임 탐색과 참여 과정을 하나의 서비스로 통합

### 발표 포인트
- 이 프로젝트는 운동을 함께할 사람을 찾는 과정의 불편함을 줄이는 데서 시작했습니다.
- 게시글만 올리고 끝나는 구조가 아니라 참여 이후 운영과 안전성까지 고려했습니다.

## 3. 문제 정의 1

### 슬라이드 제목
기존 방식의 한계

### 슬라이드 본문
- 운동 모임 정보가 카페, 오픈채팅, SNS, 커뮤니티 게시판 등으로 분산됨
- 원하는 지역과 시간대에 맞는 모임을 찾기가 어려움
- 모집 상태나 진행 상태를 신뢰하기 어려운 경우가 많음

### 발표 포인트
- 사용자는 정보가 많아도 실제로 참여 가능한 모임을 찾기가 어렵습니다.
- 서비스 기획의 출발점은 정보 부족이 아니라 정보의 정리 부족이었습니다.

---

## 4. 문제 정의 2

### 슬라이드 제목
참여 이후의 운영 문제

### 슬라이드 본문
- 모임 신청 이후 승인 여부를 한 번에 확인하기 어려움
- 내가 참여 중인 일정과 상태를 별도로 기억해야 함
- 비매너 사용자나 허위 정보에 대한 대응 구조가 약함

### 발표 포인트
- 단순 탐색 기능만 잘 만들어도 서비스는 불완전합니다.
- 실제 사용을 지속하게 만드는 요소는 참여 이후 관리 경험이라고 판단했습니다.

## 8. 사용자 시나리오

### 슬라이드 제목
대표 사용자 시나리오

### 슬라이드 본문
1. 사용자가 로그인한다.
2. 홈에서 인기 모임과 신규 모임을 확인한다.
3. 모임 찾기에서 조건에 맞는 모임을 탐색한다.
4. 상세 페이지에서 일정과 위치를 확인하고 참여한다.
5. 내 활동에서 승인 여부와 일정을 확인한다.
6. 필요 시 신고하거나 리뷰를 남긴다.

### 발표 포인트
- 서비스는 사용자가 한 번 방문하고 떠나는 구조가 아니라 반복적으로 확인하는 흐름을 목표로 했습니다.
